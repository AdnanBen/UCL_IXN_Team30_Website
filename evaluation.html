Known Bugs


  • Desktop capture may show a gray screen until a frame update happens on it e.g. user moves mouse onto that screen.

  • Desktop configuration sub-menu may show an additional monitor is present when physically there is not.

  • If the user is using a laptop with both integrated graphics and a dedicated graphics processing unit, if the unity program is run by windows using the dedicated GPU, then desktop capture will fail to work. To circumvent in such cases the user must set in windows settings for the program to be run on integrated graphics.

Critical Analysis


User interface / user experience

The user interface is simple and intuitive, with guided setup through the stages of configuring its use. Saving and loading camera angles could be improved by adding some audio or visual feedback when an action is executed, as it is controlled by the user entirely via keyboard hotkeys.

The current solution requires 3 parts: web app, OBS, Unity program. While OBS was part of our project description and required by the clients, we can evaluate the user experience of this system - For transition to a widely used program it might be necessary to merge parts together, but will come at the expense of flexibility. If OBS dependency is removed, capability of its powerful features will be lost and will have to be made up for in the standalone application. Removing OBS will allow video conferencing to be built in the Unity program, e.g. using chromium embedded framework. Overall we feel that having flexibility and strong feature capability is more important.


Functionality

Our program fills a void in the virtual presentation software landscape, it can be a valuable tool for at-home presentation broadcasts.

With high quality, customizable virtual environments, our Unity program provides a good return on the time investment needed to setup and use the program, as opposed to just using a static image of an environment in OBS using its built-in desktop and window capture.

Desktop capture inside scenes was a strong feature of our program. It could be improved by allowing capture of individual windows instead of the entire desktop, we did not look into this due to time constraints.

Algorithmic background removal in the web app is another highlight feature, we are very happy with how well it works, even producing better results than some commercial software we looked at during research.


Stability

As our desktop program is built with Unity game engine, it has good stability. There were no crashes or stability issues experienced during development and testing.

The web app has great stability as the video streaming is peer-to-peer, therefore it can withstand potential server issues.


Efficiency

The Unity program and web app were tested to run well on hardware of ranging power. A vSync frame limit was set on the Unity program to run at 30 frames per second, so that the user’s hardware is not being pushed to produce more frames per second than is necessary in our use case.

Similarly, the web app’s algorithmic background removal’s green screen is limited to around 120ms per each frame (~8 fps) to reduce load on the user’s system.


Compatibility

The web app was fully tested to work on Chrome, but it should be compatible on all modern browsers.

While our clients only expected our Unity program to work on Windows, given Unity’s cross compatibility it is possible for it to be extended in the future to support Mac and Linux. We did not do this as there are some difficulties such as platform-specific APIs for desktop capture.


Maintainability

The project is well documented in our READMEs, deployment and user manuals, code naming style and comments. The desktop program is built using a Long Term Support (LTS) version of Unity game engine.


Project management

With 3 members in the team, we were well coordinated with weekly meetings in which we planned our next steps. We also stuck to our Gantt chart quite closely to insure we never fell behind with out deliverables. Work was delegated based on strengths and previous experience.
