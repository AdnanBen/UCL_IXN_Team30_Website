Related Projects Review

Our project, as requested by our clients, was designed to break away from the current virtual environment standard. A summary of software leading in the virtual environment industry and reflecting the current standard can be found below:

VRChat

A multiplayer online game where users can explore virtual worlds together.

Key features:

- VR support
- In-game voice communication between users
- 3D spatialized audio
- Full body avatars with lip syncing, eye and full body tracking
- Creating customised worlds


Bigscreen

Bigscreen allows users to hang out and watch media together in a virtual environment.

Key features:

- VR support
- In-game voice communication between users
- Streaming desktop into game
- Native video player for streaming media


As demonstrated, these projects are quite different from ours. In our project a user being is able to place themselves in a 3D environment and project the resulting feed onto a conferencing application. In contrast, these applications provide a platform where users can login to a world, walk around inside it and communicate specifically within the environment.

However, we are still able to draw some helpful conclusions to guide us in our project. First of all, although Bigscreen lacks many immersive features in comparison to VRChat such as full body tracking and specialized audio, however it still appears to be the popular choice when the goal is simply watching media together. This suggests users prefer a specialised application for specialised tasks suggesting we should not chase immersive, irrelevant features when designing our application and instead focus on the core functionalities. Additionally, support for VR seems quite popular with most users opting to use this method of interaction rather than an ordinary mouse/keyboard monitor setup. This suggests a VR feature for our application may be highly sought after.

Microsoft Teams "Together Mode"

A mode in teams which places all meeting participants onto a 2D picture.

Key features:

- 4 or more participants
- 2D static background
- Not customisable

The presence of this service implies a demand for what we aim to create. Although the service differs in that both meeting participants in addition to presenters are seen within the scene rather than just presenters, it still somewhat resembles our goal. We plan to build on the mode by providing 3D, customizable environments with interactive scene components to provide a more interesting and engaging platform for virtual presentations.

Technology Review

Due to the nature of our project, many of the tools and technologies were defined to us by the client. However particular areas such as our webapp and certain requested functionalities required much research and consideration.

Multiple presenters

We initially considered sharing the videos of presenters with the main presenter using a python application. For this we looked into using sockets. After building a prototype however we concluded that requiring users of the software to port forward in order to allow the socket connection to be established as too high a cost.
Our second approach involved Javascript and WebRTC. Although requiring a web server for authentication, this approach would not require any port forwarding on the user’s end. By default WebRTC allowed for one to one connections however using the PeerJS library we created an implementation that allows several users to connect together in a peer to peer nature. This meant that in addition to the web server not requiring much bandwidth, in the case of a server outage, connected clients would remain connected.

Background removal

We initially looked into the “Real-Time High-Resolution Background Matting” paper as a possible solution to our automatic background removal. We created an initial python prototype which used this implantation. After further analysis we concluded that the requirement of a dedicated GPU as well as the poor frame rate in the prototype makes this solution inadequate.
Our next approach was to use Google’s BodyPix library. We created a python application utilising the library and had promising performance results. As a result, we rewrote the implementation in JavaScript and incorporated it into our multiple presenters web app.
